# JSONify 项目架构文档 V1.0

## 项目概述

JSONify 是一个基于 React + TypeScript 的纯前端 JSON 解析工具，提供 JSON 验证、格式化、压缩、转义、Unicode 转换等功能。项目采用现代化的前端技术栈，具有良好的代码组织结构和可维护性。

## 技术栈

### 核心技术
- **React 19**: 前端框架，使用函数组件和 Hooks
- **TypeScript**: 类型安全的 JavaScript 超集
- **Vite 7.x**: 现代化构建工具，提供快速的开发体验
- **Ant Design 5.x**: UI 组件库，提供丰富的组件和主题支持

### 关键依赖
- **Monaco Editor**: VSCode 同款编辑器，提供语法高亮和错误提示
- **@monaco-editor/react**: Monaco Editor 的 React 封装

## 架构设计

### 1. 分层架构

项目采用典型的前端分层架构：

```
┌─────────────────────────────────────┐
│           UI Layer (组件层)           │
│  App.tsx, JsonEditor, ErrorDisplay  │
├─────────────────────────────────────┤
│        Business Layer (业务层)       │
│     useJsonParser, useLocalStorage   │
├─────────────────────────────────────┤
│         Utils Layer (工具层)         │
│  jsonUtils, unicodeUtils, storage    │
├─────────────────────────────────────┤
│         Types Layer (类型层)         │
│        TypeScript 类型定义           │
└─────────────────────────────────────┘
```

### 2. 组件架构

#### 主应用组件 (App.tsx)
- **职责**: 应用的主入口，负责整体布局和状态管理
- **特点**: 
  - 使用 Ant Design 的 Layout 组件构建页面结构
  - 集成多个自定义 Hook 管理状态
  - 采用三栏布局：输入区(50%) | 输出区(35%) | 历史记录(15%)

#### 核心组件

1. **JsonEditor 组件**
   - 基于 Monaco Editor 的 JSON 编辑器
   - 支持语法高亮、错误提示、代码折叠
   - 可配置主题、只读模式、行号显示等

2. **ErrorDisplay 组件**
   - 错误信息展示组件
   - 支持错误位置定位和上下文显示

3. **HistoryPanel 组件**
   - 历史记录管理面板
   - 支持历史记录的查看、重命名、删除

4. **ThemeToggle 组件**
   - 主题切换组件
   - 支持明暗主题切换

### 3. Hook 架构

#### useJsonParser Hook
- **核心业务逻辑**: 封装所有 JSON 处理功能
- **状态管理**: 管理输入文本、输出文本、错误信息等状态
- **功能方法**: 提供验证、格式化、压缩、转义等方法

```typescript
interface JsonParserState {
  inputText: string;           // 输入文本
  outputText: string;          // 输出文本
  validationResult: JSONValidationResult; // 验证结果
  error: ErrorInfo | null;     // 错误信息
  isProcessing: boolean;       // 处理状态
}
```

#### useLocalStorage Hook
- **数据持久化**: 封装 localStorage 操作
- **自动保存**: 支持自动保存和手动保存
- **类型安全**: 提供泛型支持，确保类型安全

## 编程逻辑与设计模式

### 1. 状态管理模式

项目采用 **Hook + Context** 的状态管理模式：

```typescript
// 使用自定义 Hook 封装业务逻辑
const {
  inputText, outputText, error,
  validate, format, compress,
  // ... 其他方法
} = useJsonParser();

// 使用 useLocalStorage Hook 管理持久化状态
const [settings, setSettings] = useLocalStorage<UserSettings>(
  'json_parser_settings', 
  defaultSettings
);
```

### 2. 错误处理模式

采用 **Try-Catch + 用户友好提示** 的错误处理模式：

```typescript
const validate = useCallback(() => {
  setState(prev => ({ ...prev, isProcessing: true }));
  
  try {
    const result = validateJSON(state.inputText);
    setState(prev => ({
      ...prev,
      validationResult: result,
      error: result.isValid ? null : {
        message: result.error || 'Invalid JSON',
        position: result.position,
        context: state.inputText
      },
      isProcessing: false
    }));
  } catch (error) {
    setState(prev => ({
      ...prev,
      error: {
        message: error instanceof Error ? error.message : 'Validation failed'
      },
      isProcessing: false
    }));
  }
}, [state.inputText]);
```

### 3. 组件通信模式

采用 **Props Down, Events Up** 的通信模式：

```typescript
// 父组件向子组件传递数据和回调
<JsonEditor
  value={inputText}
  onChange={setInputText}
  theme={settings.editorTheme}
  onError={handleError}
/>

// 子组件通过回调函数向父组件传递事件
const handleHistorySelect = useCallback((item: HistoryItem) => {
  setInputText(item.input);
  setOutputText(item.output);
  message.success('已加载历史记录');
}, [setInputText, setOutputText]);
```

### 4. 性能优化模式

#### 防抖优化
```typescript
// 自动保存使用防抖，避免频繁写入
useEffect(() => {
  if (settings.autoSave) {
    const timer = setTimeout(() => {
      localStorage.setItem('json_parser_input', inputText);
    }, 500); // 500ms 防抖
    return () => clearTimeout(timer);
  }
}, [inputText, settings.autoSave]);
```

#### 回调优化
```typescript
// 使用 useCallback 避免不必要的重渲染
const copyToClipboard = useCallback(async (text: string) => {
  try {
    await navigator.clipboard.writeText(text);
    message.success('已复制到剪贴板');
  } catch (error) {
    message.error('复制失败，请手动复制');
  }
}, []);
```

项目全面使用 TypeScript，确保类型安全：

interface UserSettings {

export const smartUnicodeConvert = (text: string): string => {
  // 检测文本类型，自动选择转换方向
  if (containsUnicode(text)) {
    return unicodeToChinese(text);  // Unicode → 中文

  }

### 2. 自动保存机制

```typescript
export class AutoSaveManager {
  private timer: ReturnType<typeof setTimeout> | null = null;
  
  public triggerAutoSave(saveFunction: () => void): void {
    // 清除之前的定时器
    if (this.timer) {
      clearTimeout(this.timer);
    }
    
    // 设置新的定时器
    this.timer = setTimeout(() => {
      saveFunction();
      this.timer = null;
    }, this.delay);
  }
}
```

  }));
}, []);
